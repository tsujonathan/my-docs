# URL-Shortener Rate Limit Domain Override Investigation

## What We Observed

Both `url-shortener-gateway` and `services-gateway` received the same domain parameter override
for the Envoy rate limit service, even though the intent was to only override the
url-shortener's own listener.

## Codebases Involved in the Setup

| # | Codebase | Role |
|---|----------|------|
| 1 | `TERRAFORM-PLATFORM-PROVISIONING` | Drives `DS-ISTIO-RESOURCES` Helm values; controls per-gateway `istioLabel` and rate limit enablement per cluster |
| 2 | `DS-ISTIO-RESOURCES` | Provisions Gateway CRDs, TLS, base `envoy.filters.http.ratelimit` HTTP filter per gateway |
| 3 | `MSF-CHART` | Generates `grl-dom-*` and `grl-svc-*` EnvoyFilters based on `rateLimiting.global.gateway` values |
| 4 | `URL-SHORTENER` | Defines VirtualServices and `rateLimiting.global.gateway` config for the resolver service |

## Root Cause — Four Conditions All Present Simultaneously

### 1. Route name collision *(owned by `URL-SHORTENER`)*

The url-shortener has two VirtualServices — one bound to `services-gateway` and one bound to
`url-shortener-gateway`. Both define a route named `resolver`. The names are identical across
both VirtualServices.

### 2. Broad `vhost.name: ""` *(owned by `MSF-CHART`)*

The `grl-dom-*` EnvoyFilter generated by `MSF-CHART` uses `vhost.name: ""` in the route match,
which matches **all virtual hosts across all listeners** on the targeted pod — not just the
url-shortener's listener.

### 3. Shared pod label *(owned by `URL-SHORTENER`)*

The url-shortener's `rateLimiting.global.gateway.gatewayName` is set to `ingressgateway` — the
shared pod label used by all gateways including `services-gateway`. Both `url-shortener-gateway`
and `services-gateway` run on the same Envoy proxy pod.

### 4. No dedicated pod label for `url-shortener-gateway` *(owned by `TERRAFORM-PLATFORM-PROVISIONING` and `DS-ISTIO-RESOURCES`)*

No `istioLabel` override is set for the `url-shortener` gateway in
`TERRAFORM-PLATFORM-PROVISIONING`'s values. `DS-ISTIO-RESOURCES` then defaults the Gateway CRD
selector to `istio: ingressgateway`, placing it on the shared pod.

## How the Conditions Combine

The combination of **(2)** and **(3)** means the EnvoyFilter targets all listeners on the shared
pod. The combination of **(1)** then means the route name match hits the `resolver` route on
**both** listeners — the one on `url-shortener-gateway` (intended) and the one on
`services-gateway` (unintended).

This is not a misconfiguration by the url-shortener team — all values are set as expected given
the current chart design. The collision is a consequence of route name reuse being the only guard
when `vhost.name: ""` is combined with a shared pod label.

## Recommended Fixes

### Fix 1 — MSF-CHART (preferred, systemic fix)
Change the EnvoyFilter template to use `vhost.name: <firstGatewayDomain>` instead of
`vhost.name: ""`, matching the SNI-scoped approach already used by `DS-ISTIO-RESOURCES`'s
`rate-limit-filter.yaml`. This eliminates the collision class entirely regardless of route name
reuse.

### Fix 2 — URL-SHORTENER (immediate workaround)
Rename the route on the `additionalVirtualService` (the one bound to `url-shortener-gateway`)
from `resolver` to something unique like `url-shortener-resolver`. Then update
`rateLimiting.global.gateway.routeName` in their values to match. This breaks the collision with
no infrastructure changes.

### Fix 3 — TERRAFORM-PLATFORM-PROVISIONING + DS-ISTIO-RESOURCES (cleanest isolation)
Provision a dedicated Envoy pod for `url-shortener-gateway` by adding
`istioLabel: "ingressgateway-url-shortener"` to the Terraform values for that gateway. Then set
`gatewayName: "ingressgateway-url-shortener"` in the url-shortener's
`rateLimiting.global.gateway` config. This fully isolates it but requires a new pod deployment.

## Production Risk

The unintended effect is that traffic hitting a route named `resolver` on `services-gateway`
(`*.services.dev.docusign.net`) is subject to the url-shortener's rate limit domain counters in
the Rate Limit Service — meaning url-shortener's quota could be exhausted by `services-gateway`
traffic and vice versa. Given the `ratelimit-limit: 1000/hour` observed, this is a real
production risk worth prioritizing.
